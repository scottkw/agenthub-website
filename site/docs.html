Documentation

Everything you need to run, attach, and share AI coding sessions with AgentHub.

Working on v3.3 (May 2026)These docs track the latest release. For older releases, use the version switcher on GitHub.

Quickstart

AgentHub is a single binary. Run it with no arguments to open the desktop GUI; pass a subcommand to drive the CLI or TUI. All three surfaces talk to the same background daemon.

Install

PlatformInstallNotes
macOSbrew tap scottkw/agenthub && brew install --cask agenthubSigned + notarized universal
Linux.deb or .tar.gz from ReleasesUbuntu 22.04 & 24.04
Windowswinget install scottkw.agenthubNSIS installer or standalone .exe

Your first session

# launch the GUI — click "New Session" in the sidebar
agenthub

# or go straight to a terminal session
agenthub new claude-code ~/projects/api

# list everything running across your machines
agenthub list

Agent CLIs (Claude Code, Codex, Gemini CLI, OpenCode) are auto-detected across common install locations on startup — nvm, Volta, Homebrew, snap, flatpak, cargo, pipx, and platform-specific paths. If an agent isn't found, override its path in Settings → Paths. A raw shell (bash / zsh / fish / PowerShell / cmd.exe) is always available too — see Shell sessions.

Desktop GUI

The Wails-based desktop app gives you tabbed terminals, a collapsible sidebar, and in-app settings. Closing the window hides it to the tray — sessions keep running in the daemon.

Sidebar

  • Home — welcome tab with install status & version
  • Remote — sessions on other tailnet peers
  • Sessions — all active sessions on this machine
  • New Session — opens the CLI picker + folder browser
  • Settings — appearance, web server, paths, behavior

Tabs & terminals

Every open session shows a colored status dot in its tab. Double-click a tab to rename it, or right-click for a context menu. Shift+= and Shift+- zoom per-tab (6–32px).

Auto-close on exitWhen an agent exits cleanly, a 5-second countdown appears with an inline banner and toast. Click Keep Open to cancel. Non-zero exits stay open and show the error state. Toggle this in Settings → Session Behavior.

CLI reference

Every CLI command also accepts --json for machine-readable output (where applicable).

Sessions

agenthub new <agent> <dir> [-- extra-args]
agenthub list [--json]
agenthub attach <id> [--readonly] [--client=<name>] [--detach-key=C-\]
agenthub attach hostname:<id>        # remote via tailnet
agenthub kill <id>
agenthub rename <id> "my session"

TUI

agenthub tui                        # full-screen Bubble Tea UI

Web serving

agenthub web start | stop | status
agenthub serve <id>                 # enable web access
agenthub unserve <id>
agenthub qr <id>                    # show session QR in terminal
agenthub health                     # Tailscale health cascade

Daemon

agenthub daemon install | uninstall | start | stop | status

Configuration

agenthub settings                   # show current settings

Attach keys

KeyAction
Ctrl+\Detach (configurable with --detach-key)

During attach, a tmux-style status bar pins session name, agent, hostname, elapsed time, and live viewer count to the bottom of the terminal. Pass --status-top to flip it. The bar suppresses itself when stdout isn't a TTY.

TUI mode

agenthub tui launches an interactive Bubble Tea v2 interface with near-GUI parity. It's two panes: a sidebar on the left, content on the right. Tab toggles focus; [ and ] cycle tabs.

KeyAction
↑ ↓Navigate list
EnterAttach to selected session
nNew session modal (agent picker + dir)
dKill session (with danger-styled confirmation)

qShow ASCII QR overlay

Sessions render inside bordered lipgloss frames with colored per-agent badges. Attach suspends the TUI and enters raw PTY mode; Ctrl+\ returns you. The session list auto-refreshes every 2 seconds with selection preserved by identity.

Shell sessions v3.3

Alongside the four AI CLIs, AgentHub runs raw PTY shell sessions — bash, zsh, fish, dash, powershell.exe, pwsh, or cmd.exe. Every surface exposes this as exactly one entry labelled “Shell”; there is no per-surface picker for the spawned binary. The daemon resolves it from a single Settings value, so the same shell launches from the GUI, TUI, and CLI.

Launch a shell

# GUI — open New Session, pick the "Shell" row
agenthub

# TUI — press n, cycle the agent picker to "Shell"
agenthub tui

# CLI — optional positional working directory; omit to launch in $HOME
agenthub new shell [<path>]

Extra tokens after -- are not forwarded to shell sessions (a stderr warning is emitted if present, mirroring the GUI's no-args behaviour). The CLI has no --shell flag — selection happens once, in Settings.

Choose the shell binary

Open Settings → Paths → Shell binary path and pick the executable. The native folder browser is the same one used for CLI overrides. Leave it blank to fall back to $SHELL (or the platform default: zsh on macOS, bash on Linux, powershell.exe on Windows). Discovery surfaces canonical install locations on every platform — on Windows that includes C:\Program Files\PowerShell\7 and %LOCALAPPDATA%\Microsoft\WindowsApps so PowerShell 7 is picked up wherever it landed.

Web sharing a shell is opt-in, with a bannerShells expose your full command-line surface — far broader than the focused output of an AI CLI. Toggling web share on a shell session raises a ShellWebShareBanner that surfaces the risk and asks you to confirm. The acknowledgement persists per machine, so you only see the banner the first time per install.

Status heuristics specific to AI CLIs are bypassed for shells, so a shell tab simply stays running until you exit it. When the shell process exits cleanly (status code 0), the standard auto-close flow applies — 5-second countdown, Keep Open cancel — with PTY exit-code -1 normalized to 0 so a tidy exit never falls through to the “errored” branch.

Multi-client sessions

The relay hub fans one PTY's output to any number of connected clients over WebSocket. Every client gets its own scrollback position, font size, and identity.

  • Read-only observers — agenthub attach --readonly <id>
  • Named clients — agenthub attach --client=macbook <id>
  • Resize arbitration — PTY dimensions stabilize to the largest active client (max-wins)
  • Viewer count — visible in the status bar, agenthub list, and the GUI Sessions panel

Remote & Tailscale

When Tailscale is running, AgentHub auto-discovers other AgentHub peers on your tailnet and runs the web server over HTTPS with Let's Encrypt certificates — no port forwarding, no self-signed warnings. Without Tailscale, the web server falls back to self-signed TLS on your local network with HTTP Basic Auth using a generated password from Settings → Web Server. AgentHub upgrades automatically to Tailscale mode once it detects a connected daemon.

Sharing is explicit (v3.1+)The web server runs by default, but individual sessions are not auto-exposed. Each share is a deliberate action — toggle web access on a session (GUI/CLI) and AgentHub mints an HMAC-signed, session-bound capability token. The returned share URL carries that token as ?cap=<...>; QR codes encode a short /join?code=XXXX-XXXX exchange so leaked screenshots never grant access. Read-only is enforced server-side from the token's claims, not the query string. Use Settings → Security → Regenerate Signing Key as a panic button to invalidate every outstanding capability across all sessions.
# list remote sessions across your tailnet
agenthub list
# HOST             ID       AGENT   NAME       VIEWERS
# scottkw-mbp      a1b2c3   claude  payments   2
# laptop.ts.net    d4e5f6   gemini  docs       0

# attach to a remote session (relays over WSS)
agenthub attach laptop.ts.net:d4e5f6
Health cascadeRun agenthub health for a 4-state check: binary found → daemon running → connected → TLS certs ready. Platform-specific troubleshooting is surfaced inline when a step fails.

AgentHub ships with 138 WCAG-audited terminal themes from the xterm-theme library. Pick one in Settings → Appearance; it applies live to every open terminal (including OpenCode, which honors theme changes via SIGUSR2 broadcast) and persists across restarts.

Terminal plugins (v3.2)

The terminal core is extended by a curated, vendored xterm.js addon suite — eight plugins bundled under web/vendor/xterm/addons/ with a CI-enforced version-parity gate. Zero runtime CDN fetches, fully CSP-compatible across Chromium, Firefox, and WebKit. Toggle each from Settings → Plugins; daemon settings are the source of truth and hot-swap to live sessions where possible.

Sub-key changes persist immediately — no "Save Plugins" required. Cross-browser e2e enforces zero CSP violations in CI (Chromium + Firefox + WebKit via Playwright).

Daemon management

The daemon runs as a background process owning all PTYs, the relay hub, and the web server. Communication happens over a Unix socket (macOS/Linux) or named pipe (Windows).

Register AgentHub as a login service so sessions survive reboots:

agenthub daemon install        # registers with launchd / systemd / Windows SCM
agenthub daemon status
agenthub daemon uninstall

Settings

Settings live as a single scrollable page in the sidebar — no modals, no sub-tabs. A hyperlinked index sits at the top of the page so you can jump straight to any section.

  • Plugins (v3.2) — 8 toggles for the xterm.js addon suite, with inline disclosures for Search / Web Links / Inline Images sub-config; hot-swap for runtime plugins, "Applies to new sessions" inline for the rest
  • Appearance — 138 terminal themes, live preview
  • Web Server — start/stop, mode-aware status, URL actions (open / copy / inline QR), local-network password with click-to-copy
  • Security (v3.1) — Regenerate Signing Key panic button (invalidates every outstanding capability across all sessions), Tailscale 4-state status indicator with collapsible diagnostics, Certificate Transparency disclosure
  • Paths — override auto-detected CLI binaries with a native folder picker; v3.3 adds a Shell binary path row that drives shell sessions across GUI / TUI / CLI
  • Behavior — start minimized to the system tray
  • Session Behavior — auto-close tab on clean exit

Troubleshooting

Agent CLI not detected

AgentHub searches nvm, Volta, Homebrew, snap, flatpak, cargo, pipx, and platform default paths on startup — including when launched from Finder. If yours is missing, add its absolute path in Settings → Paths.

Tailscale mode not activating

Run agenthub health. It surfaces exactly which of the 4 checks (binary / daemon / connected / certs) is failing, with platform-specific instructions. AgentHub auto-upgrades to Tailscale mode once the daemon connects.

Sessions still running after GUI close

Correct — that's the design. Closing the window hides the app; the daemon stays alive. Use the Quit Confirmation modal (or the tray menu) to fully stop everything.
